Creando variables y constantes

Creando variables con var

  • No puede empezar con un número
  • No puede empezar con caracteres especiales
                            
                                var nombre='Jesus', apellido='Chuye';
                                var nombre='Kenny', apellido='Chuye Duque';//puede asignársele otro valor
                                var nombreCompleto = nombre + " " + apellido;

Creando variables con let

let no permite la redeclaración de una variable.

                            
                                let curso='Javascript';
                                let curso='PHP'; //esto arrojará error

Si permite la reasignación de un valor.

                                
                                    let curso='Javascript';
                                    curso='PHP'; //esto NO arrojará error

Si permite la declaración inicial en blanco o vacio para la posterior asignación de un valor.

                                
                                    let curso=' ';
                                    curso='PHP';

En conclusión se puede usar let cuando posteriormente se piensa reasignar un valor.

Creando variables con const

const no permite la reasignación de un valor

                            
                                const pi='3.1416';
                                const igv=18;

No permite la reasignación de un valor

                            
                                const curso='Javascript';
                                curso='PHP'; //esto arrojará error.

No permite la inicialización en blanco para la posterior asignación de un valor.

                            
                                const curso=' ';
                                curso='PHP'; //esto arrojará error.

En conclusión se puede usar const cuando el valor no se piensa cambiar NUNCA.

Convertir string a número

                            
                                const num1='50', num2=10 num3='tres';
            
                                console.log(num1 + num2); // devolverá 5010
                                console.log(num1 - num2); // devolverá 40
            
                                //Number() => convierte un string en número
            
                                console.log(Number(num1) + num2); // devolverá 60
                                
                                console.log(Number(num3)); // devolverá NaN (no es un número)
                                
                                console.log(Number(true)); //devolverá 1
            
                                console.log(Number(false));//devolverá 0
                                
                                console.log(Number(null));//devolverá 0
                                
                                console.log(Number(undefined));//devolverá NaN
            
                                //parseInt() también convierte un string en número
            
                                console.log(parseInt(num1) + num2); // devolverá 60
            
                                console.log(parseInt('100.203'); // devolverá 100 como número
                                
                                console.log(parseFloat('100.203'); // devolverá 100.203 como número.
                                
                                // toFixed()
            
                                const dato=18929173;
                                console.log(dato.toFixed()); // devolverá 18929174.
                                
                                const dato=18929173.9812912871371193899;
                                console.log(dato.toFixed()); // devolverá 18929174.
                                console.log(dato.toFixed(3)); // devolverá 18929173.981

API's, REST API's y Request

API = Application Programming Interface

  • Son funciones, métodos que ofrece una librería para ser utilizada por otro software como una capa de abstración.
  • Para acceder al servicio hay que enviar una petición estructurada.
  • Ejemplo: Google Maps API

REST API's = Representational State Transfer

  • Puede ser diseñada en cualquier lenguaje.
  • REST describe como deben ponerse a disposición los recursos.
  • La API debe responder a los Request de HTTP como GET, POST, PUT, PATCH, DELETE.
  • Cada API tiene sus propias reglas, métodos, estructuras.

REST API EndPoints y Request.

  • Una REST API cuenta con EndPoints ( o Urls) para hacer operaciones CRUD.
  • Por ejemplo para Listar todos los clientes el formato sería: Request = GET     |     EndPoint = /clientes.
  • Si sólo se desea un solo cliente el formato sería: Request = GET     |     EndPoint = /clientes/id_del_cliente .
  • Para crear un nuevo cliente: Request = POST     |     EndPoint = /clientes.
  • Para editar un contacto: Request = PUT     |     EndPoint = /clientes/id_del_cliente.
  • Borrar un contacto: Request = DELETE     |     EndPoint =/clientes/id_del_cliente.

Códigos Síncrono y Asíncrono en Javascript

En el Código Síncrono:

  • En un programa de 2 líneas, la línea 2 no puede empezar sin haber terminado la línea 1.
  • Ejemplo: La fila para pagar del supermercado.

En el Código Asíncrono:

  • En un programa de 2 líneas, la línea 2 puede ejecutarse y finalizarantes que la línea 1 lo haga.
  • Ejemplo: Un restaurant.

En Javascript:

  • Cuando se consume APIs O REST APIs, será cuando se esté ejecutando Javascript Asíncrono.
  • XMLHTTPRequest y Fetch API son ejemplos de código asíncrono.
  • jQuery es un ejemplo de una líbrería de este tipo.

Las siguientes son funciones para crear código Asíncrono:

  • Callbacks.
  • Promises.
  • Async / Await.

Destructurando:

                                     nombre: 'Alejandro',
                                     tipo: 'Premium'
                                }
        
                                let {nombre, tipo} = cliente;

Spread Operator

Es un modificador que permite simplificar operaciones entre objetos y/o arrays, permitiendo agilizar sus procesos

                            
                                const Lista1 = [1,2,3]
                                const Lista2 = [4,5,6]

                                //Aplicando spread operator
                                const ListaTotal = [...Lista1, ...Lista2]
                                console.log(ListaTotal);
                            
                        

                            const Lista1 = [1,2,3]
                            const Lista2 = [4,5,6]
                            
                            function suma(valor1, valor2, valor3){
                                let suma= valor1 + valor2 + valor3;
                                console.log(suma);
                            }
                            
                            suma(...Lista1);

Spread Operator con Objetos

Permite unir fácilmente una colección de objetos y sus propiedades. Cuando distintos objetos tienen una misma o varias propiedades iguales sólo se tomará en cuenta la última propiedad.


                            const perfil = {
                            nombre: 'Gerardo',
                            info: {
                            direccion: 'Calle 100'
                            }
                            }
                            
                            const region = {
                            pais: 'México',
                            info: {
                            coordenas: 'Norte 500 - Sur 200'
                            }
                            }
                            
                            const social = {
                            twitter : '@luxfenix',
                            nombre: 'Ninja Pro'
                            }
                            
                            const resultado = {
                            ...perfil,
                            ...region,
                            ...social
                            }
                            
                            console.log(resultado);

Sin embargo, la solución es un spread operator dentro de otro spread operator


                            const resultado = {
                            ...perfil,
                            ...region,
                            ...social,
                            info: {
                            ...perfil.info,
                            ...region.info
                            }
                            }
                            
                            console.log(resultado);

Rest Operator

Es la función inversa al spread operator.

Recibe un conjunto de elementos separados por comas y los une en un array.


                            const Lista1 = [1,2,3]
                            const Lista2 = [4,5,6]
                            
                            function suma(...numeros){
                                let suma = numeros.reduce((a, b) => a +b);
                                console.log(suma);
                            }
                            
                            suma(...Lista1);
                            suma(...Lista1, ...Lista2);
                            suma(14, 18, 47, 100, 9, 32, 16);

Rest Operator con Objetos


                            
                            const personaje = {
                                nombre: 'Luis',
                                apellido: 'Castillo',
                                alergias: {
                                    trigo: true,
                                    lactosa: false
                                }
                            }
                            
                            const direccion = {
                                calle: 'Calle Falsa 123',
                                localidad: 'Paris',
                                pais: 'Francia'
                            }
                            
                            const persona = {
                                ...personaje,
                                ...direccion,
                                edad: 45
                            }
                            
                            console.log(persona);

El objeto copia resultante persona es una copia por referencia (de los objetos personaje y direccion) y no una copia de valor.
Esto quiere decir que si al objeto personaje se le cambia el valor de una de sus propiedades, éste no afectará al objeto copia persona.


                            personaje.nombre = 'Jesús';
                            console.log("Objeto personaje");
                            console.log(personaje);
                            console.log("===================");
                            console.log("Objeto copia persona");
                            console.log(persona);

Sin embargo, si se hace un cambio de valor a una propiedad de un subnivel (un objeto dentro de otro objeto); este cambio si se hará efectivo en el objeto copia resultante persona 


                            personaje.alergias.lactosa = true
                            console.log("Objeto personaje");
                            console.log(personaje);
                            console.log("===================");
                            console.log("Objeto copia persona");
                            console.log(persona);

Esto se debe a que Object.assign no itera en profundidad, esto quiere decir que, sólo va a revisar las propiedades de primer nivel, para las de segundo nivel sólo será tomada en cuenta la última, por ejemplo, en el caso de un ojeto dentro de otro objeto.

                                }
                            }

                                {},
                                perfil,
